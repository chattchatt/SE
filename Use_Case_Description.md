
## <허수민 팀원>
### Use case description : 자전거 등록
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
|1. 관리자가 자전거 등록 창에 접근한다.|2. 자전거 정보(자전거 ID, 자전거 제품명, 유형(일반/전기), 소속 대여소, 상태(사용 가능/수리 중) 등)를 입력할 수 있는 화면을 보여준다.|
|3. 관리자가 자전거 정보(자전거 ID, 자전거 제품명, 유형(일반/전기), 소속 대여소, 상태(사용 가능/수리 중) 등)를 입력하고 등록 버튼을 클릭한다.|4. 정상적으로 저장되었음을 알려준다.|

## <김수림 팀원>
### Use case description : 대여소 등록
| Actor Action                                                  | System Response                         |
|---------------------------------------------------------------|-----------------------------------------|
| 1. 관리자는 메뉴에서 대여소 등록을 선택한다.                                    | 2. 시스템은 대여소 등록에 필요한 정보를 입력 할 페이지를 보여준다. |
| 3. 관리자는 대여소 이름, 대여소 위치 (도시, 주소), 자전거 보관 가능 수량, 운영 시간 등을 입력한다. | 4. 시스템은 대여소 등록 성공 메시지를 사용자에게 보여준다.      |

### Use case description : 대여소 리스트 조회
| Actor Action                                                    | System Response                                                                          |
|-----------------------------------------------------------------|------------------------------------------------------------------------------------------|
| 1. 관리자는 메뉴에서 대여소 리스트 조회를 선택한다.                                  | 2. 시스템은 현재 까지 등록되어 있는 모든 자전거 대여소 리스트를 새로운 페이지에서 보여준다.|

| Extensions |
|------------|
| 2a 관리자는 대여소 리스트에서 특정 대여소 옆의 삭제 버튼을 눌러 삭제 할 수 있다.    

### Use case description : 대여소 상세 내용 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 관리자는 대여소 리스트에서 원하는 대여소를 선택한다.|2. 시스템은 선택된 대여소의 상세 정보 (대여소가 등록될 때 관리자가 입력했던 정보들) 를 새로운 페이지에서 사용자에게 보여준다.

### Use case description : 회원 가입
| Actor Action                                           | System Response                                                                          |
|--------------------------------------------------------|------------------------------------------------------------------------------------------|
| 1. 시스템을 처음 이용하는 사용자는 메뉴에서 회원 가입을 선택한다.                 | 2. 시스템은 회원 가입을 위해 필요한 필수 정보를 입력 할 페이지를 보여준다.|
| 3. 사용자는 필수 입력 정보인 ID, 비밀 번호, 결제 수단, 선호 자전거 유형 등을 입력한다. |4. 시스템은 회원 가입 완료 메시지를 사용자에게 보여준다.                              

### Use case description : 회원 탈퇴
| Actor Action             | System Response                                                                          |
|--------------------------|------------------------------------------------------------------------------------------|
| 1. 고객은 메뉴에서 회원 탈퇴를 선택한다. |2. 시스템은 탈퇴 여부 확인 메세지를 보여준다.|
| 3. 고객은 '확인' 버튼을 클릭한다. | 4. 시스템은 탈퇴 성공 확인 메세지를 보여준다. |

## <전채운 팀원>
### Use case description : 자전거 대여 정보 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객은 ‘내 대여 정보’ 메뉴를 선택한다.        | 2. 시스템은 현재 대여 중인 자전거 목록을 표시한다 (대여소 이름, 대여소 위치, 자전거 ID, 자전거 제품명, 자전거 유형 포함)|

| Extensions |
|------------|
| 2a 고객은 '자전거 반납하기'를 눌러 특정 자전거를 지정된 대여소에 반납한다.  

| Extensions |
|------------|
| 2b 고객은 '식당 추천받기'를 눌러 추천 서비스에 접속할 수 있다. 

### Use case description : 대여 금액 및 횟수 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 관리자는 ‘통계’ 메뉴를 선택한다.        | 2. 시스템은 전체 대여 정보와 함께 기간별 조회 옵션(1주일/1개월/1년)을 표시한다.|
| 3. 관리자는 원하는 기간을 선택한다.          | 4. 시스템은 선택한 기간의 총 대여 금액과 대여 횟수를 표시한다.|

| Extensions |
|------------|
| 2a. 관리자는 1주 단위 조회를 선택하여 1주일 단위 대여 금액 및 횟수를 확인할 수 있다.

| Extensions |
|------------|
| 2b. 관리자는 1개월 단위 조회를 선택하여 1개월 단위 대여 금액 및 횟수를 확인할 수 있다.

| Extensions |
|------------|
| 2c. 관리자는 1년 단위 조회를 선택하여 1년 단위 대여 금액 및 횟수를 확인할 수 있다.

### Use case description : 대여 시간 및 요금 조회하기
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객은 ‘대여 시간 및 요금 조회’ 메뉴를 선택한다.        | 2. 시스템이 회원의 반납 완료된 모든 자전거의 대여 시간 및 요금을 보여준다. |

### Use case description : 과거 내역 기록 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객이 ‘이용 내역’ 메뉴를 선택한다.        | 2. 시스템이 날짜별로 정렬된 과거 대여 기록 목록을 표시한다.|
| 3. 고객이 특정 대여 기록을 선택한다.          | 4. 시스템이 선택한 대여 기록의 상세 정보(대여소, 자전거 정보, 대여 시간, 반납 시간, 요금)를 표시한다.|

| Extensions |
|------------|
| 2a 고객이 과거 대여 기록 목록의 특정 대여 기록을 '삭제하기'를 누르면, 과거 내역 기록이 삭제된다.

| Extensions |
|------------|
| 2b 고객이 대여소 기준 조회를 선택하여, 대여소 기준 과거 내역 기록 조회가 가능하다.


## <유현모 팀원>
### Use case description : 대여소 검색
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객이 대여소 검색 메뉴로 접근한다.        |2. 검색 입력창을 표시한다.|
| 3. 고객이 대여소 이름을 입력한다.   		 |4. 조건에 맞는 대여소 목록을 표시한다.|

### Use case description : 대여소 상세 정보 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객이 대여소 검색 이후 검색된 대여소 목록에서 특정 대여소를 선택한다.       |2. 시스템은 대여소의 상세 정보(이름, 위치, 사용 가능 자전거 목록)를 표시한다.|

| Extensions |
|------------|
| 2a 고객은 자전거가 남아있는 경우, 즉시 대여를 할 수 있다.

| Extensions |
|------------|
| 2b 고객은 자전거가 남아있지 않은 경우, 예약 대기 신청을 할 수 있다.

### Use case description : 대여 자전거 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객이 대여 정보를 조회한다.         |2. 대여 중인 자전거의 상세 정보를 표시한다.|  

### Use case description : 예약 대기 조회
| Actor Action | System Response                                                                          |
|--------------|------------------------------------------------------------------------------------------|
| 1. 고객이 예약대기 목록을 조회한다.        |2. 예약대기 정보(대여소 이름, 위치, 자전거 ID, 제품명, 유형 등)를 표시한다.|

| Extensions |
|------------|
| 2a. 고객은 예약 대기 조회 목록 내에 특정 예약을 취소할 수 있다.
